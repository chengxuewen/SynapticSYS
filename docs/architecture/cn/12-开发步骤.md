# 开发步骤

本章节提供基于 ROS2 的 SynapticSYS 开发流程指南，从需求到交付的完整工程化步骤，以保证持续稳定迭代与高质量交付。

## 1. 环境搭建与准备

### 1.1 ROS2 环境安装

- **安装 ROS2**：根据操作系统安装对应版本的 ROS2（推荐 Humble 或更高版本）
  - Ubuntu: `sudo apt install ros-humble-desktop`
  - macOS: 使用 Homebrew 或源码编译
  - Windows: 使用 Chocolatey 或安装程序

- **设置环境变量**：
  ```bash
  source /opt/ros/humble/setup.bash
  ```

- **安装开发依赖**：
  ```bash
  sudo apt install python3-colcon-common-extensions python3-rosdep
  ```

### 1.2 工作空间配置

- **创建工作空间**：
  ```bash
  mkdir -p ~/synapticsys_ws/src
  cd ~/synapticsys_ws
  ```

- **初始化工作空间**：
  ```bash
  colcon build --symlink-install
  ```

- **设置工作空间环境**：
  ```bash
  source install/setup.bash
  ```

## 2. 需求与设计

### 2.1 需求分析

- **功能需求**：明确系统需要实现的功能
- **非功能需求**：性能、实时性、安全性、可观测性
- **ROS2 集成需求**：确定与 ROS2 生态系统的集成点

### 2.2 架构设计

- **系统架构**：基于 ROS2 的节点和话题设计
- **通信设计**：
  - 话题（Topics）：用于发布/订阅通信
  - 服务（Services）：用于请求/响应通信
  - 动作（Actions）：用于长时间运行的任务
- **消息设计**：定义自定义消息类型（.msg 文件）
- **服务接口设计**：定义服务接口（.srv 文件）

### 2.3 原型与评审

- **ROS2 节点原型**：创建关键节点的原型实现
- **通信链路测试**：验证节点间通信
- **HMI 原型**：开发人机界面原型
- **功能评审**：基于原型进行功能评审与迭代

## 3. 工程与实现

### 3.1 创建 ROS2 包

- **创建 C++ 包**：
  ```bash
  cd ~/synapticsys_ws/src
  ros2 pkg create --build-type ament_cmake my_cpp_package
  ```

- **创建 Python 包**：
  ```bash
  cd ~/synapticsys_ws/src
  ros2 pkg create --build-type ament_python my_python_package
  ```

### 3.2 编码规范

- **C++ 规范**：遵循 ROS2 的 C++ 编码规范
- **Python 规范**：遵循 PEP 8 编码规范
- **命名约定**：
  - 包名：小写字母，使用下划线分隔
  - 节点名：小写字母，使用下划线分隔
  - 话题名：小写字母，使用斜杠分隔
  - 服务名：小写字母，使用斜杠分隔

### 3.3 实现 ROS2 节点

- **C++ 节点实现**：
  ```cpp
  #include "rclcpp/rclcpp.hpp"

  class MyNode : public rclcpp::Node
  {
  public:
    MyNode() : Node("my_node")
    {
      // 初始化节点
    }
  };

  int main(int argc, char **argv)
  {
    rclcpp::init(argc, argv);
    auto node = std::make_shared<MyNode>();
    rclcpp::spin(node);
    rclcpp::shutdown();
    return 0;
  }
  ```

- **Python 节点实现**：
  ```python
  import rclpy
  from rclpy.node import Node

  class MyNode(Node):
      def __init__(self):
          super().__init__("my_node")
          # 初始化节点

  def main(args=None):
      rclpy.init(args=args)
      node = MyNode()
      rclpy.spin(node)
      rclpy.shutdown()

  if __name__ == "__main__":
      main()
  ```

### 3.4 实现发布者与订阅者

- **发布者**：
  ```cpp
  // C++
  publisher_ = this->create_publisher<std_msgs::msg::String>("topic_name", 10);
  ```

- **订阅者**：
  ```cpp
  // C++
  subscriber_ = this->create_subscription<std_msgs::msg::String>(
      "topic_name", 10, 
      std::bind(&MyNode::topic_callback, this, std::placeholders::_1)
  );
  ```

### 3.5 实现服务与客户端

- **服务**：
  ```cpp
  // C++
  service_ = this->create_service<example_interfaces::srv::AddTwoInts>(
      "add_two_ints", 
      std::bind(&MyNode::add_two_ints_callback, this, 
                std::placeholders::_1, std::placeholders::_2)
  );
  ```

- **客户端**：
  ```cpp
  // C++
  client_ = this->create_client<example_interfaces::srv::AddTwoInts>("add_two_ints");
  ```

### 3.6 配置文件

- ** launch 文件**：用于启动多个节点
  ```xml
  <!-- launch/my_launch.py -->
  from launch import LaunchDescription
  from launch_ros.actions import Node

  def generate_launch_description():
      return LaunchDescription([
          Node(
              package='my_package',
              executable='my_node',
              name='my_node'
          ),
      ])
  ```

- **参数文件**：用于配置节点参数
  ```yaml
  # config/my_config.yaml
  my_node:
    ros__parameters:
      param1: value1
      param2: value2
  ```

## 4. 构建与测试

### 4.1 构建项目

- **构建所有包**：
  ```bash
  cd ~/synapticsys_ws
  colcon build
  ```

- **构建特定包**：
  ```bash
  colcon build --packages-select my_package
  ```

- **构建并运行测试**：
  ```bash
  colcon build --packages-select my_package --cmake-args -DBUILD_TESTING=ON
  colcon test --packages-select my_package
  ```

### 4.2 单元测试

- **C++ 单元测试**：使用 gtest
  ```cpp
  // test/test_my_node.cpp
  #include "gtest/gtest.h"
  #include "rclcpp/rclcpp.hpp"
  #include "my_package/my_node.hpp"

  TEST(MyNodeTest, TestInitialization)
  {
    auto node = std::make_shared<MyNode>();
    ASSERT_NE(node, nullptr);
  }
  ```

- **Python 单元测试**：使用 pytest
  ```python
  # test/test_my_node.py
  import pytest
  import rclpy
  from my_package.my_node import MyNode

  def test_node_initialization():
      rclpy.init()
      node = MyNode()
      assert node is not None
      rclpy.shutdown()
  ```

### 4.3 集成测试

- **启动集成测试**：
  ```bash
  colcon test --packages-select my_package --event-handlers console_direct+
  ```

- **使用 launch_testing**：
  ```python
  # test/integration_test.py
  import launch
  import launch_testing
  import launch_ros.actions

  def generate_test_description():
      node = launch_ros.actions.Node(
          package='my_package',
          executable='my_node'
      )
      return launch.LaunchDescription([
          node,
          launch_testing.actions.ReadyToTest()
      ])
  ```

### 4.4 性能测试

- **使用 ROS2 性能测试工具**：
  ```bash
  ros2 run performance_test performance_test --ros-args --params-file test_params.yaml
  ```

- **监控工具**：
  - `ros2 topic hz`：监控话题频率
  - `ros2 topic bw`：监控话题带宽
  - `ros2 node info`：查看节点信息

## 5. 部署与发布

### 5.1 构建与打包

- **创建发布包**：
  ```bash
  colcon build --packages-select my_package --cmake-args -DCMAKE_BUILD_TYPE=Release
  ```

- **创建安装包**：
  ```bash
  colcon bundle --packages-select my_package
  ```

### 5.2 部署

- **本地部署**：
  ```bash
  source install/setup.bash
  ros2 launch my_package my_launch.py
  ```

- **容器部署**：
  ```dockerfile
  # Dockerfile
  FROM osrf/ros:humble-desktop
  COPY . /workspace/src
  WORKDIR /workspace
  RUN colcon build
  CMD ["bash", "-c", "source install/setup.bash && ros2 launch my_package my_launch.py"]
  ```

- **远程部署**：使用 SSH 或部署工具将构建产物复制到目标机器

### 5.3 监控与维护

- **日志管理**：
  ```bash
  ros2 topic echo /rosout
  ```

- **节点监控**：
  ```bash
  ros2 node list
  ros2 node info /my_node
  ```

- **话题监控**：
  ```bash
  ros2 topic list
  ros2 topic echo /my_topic
  ```

- **服务监控**：
  ```bash
  ros2 service list
  ros2 service call /my_service example_interfaces/srv/Empty "{}"
  ```

### 5.4 灰度发布与回滚

- **版本管理**：使用 git 标签管理版本
- **灰度发布**：逐步部署新版本
- **回滚机制**：准备回滚脚本，以便在出现问题时快速回滚

### 6.1 文档规范

- **包文档**：每个包应有 README.md 文件
- **API 文档**：

- **架构文档**：系统架构图、组件关系图

### 6.2 代码质量管理

- **代码风格检查**：
  - C++：使用 clang-format
  - Python：使用 flake8

- **静态代码分析**：
  - C++：使用 clang-tidy
  - Python：使用 pylint

### 6.3 安全治理

- **权限管理**：ROS2 节点的权限控制
- **数据保护**：敏感数据的加密与保护
- **安全审计**：系统操作的审计日志

## 7. 持续集成与持续部署

### 7.1 CI/CD 配置

- **GitHub Actions**：
  ```yaml
  # .github/workflows/ci.yml
  name: CI
  on: [push, pull_request]
  jobs:
    build:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v2
        - name: Setup ROS2
          run: |
            sudo apt update
            sudo apt install ros-humble-desktop python3-colcon-common-extensions
        - name: Build
          run: |
            source /opt/ros/humble/setup.bash
            colcon build
        - name: Test
          run: |
            source /opt/ros/humble/setup.bash
            source install/setup.bash
            colcon test
  ```

- **GitLab CI**：类似配置

### 7.2 自动化测试

- **单元测试**：每次提交自动运行
- **集成测试**：合并请求时运行
- **性能测试**：定期运行

### 7.3 自动部署

- **开发环境**：每次提交自动部署
- **测试环境**：合并到开发分支时部署
- **生产环境**：手动触发或定期部署

## 8. 开发工具与最佳实践

### 8.1 推荐工具

- **IDE**：
  - Visual Studio Code + ROS Extension
  - CLion + ROS Support
- **调试工具**：
  - `gdb`：C++ 调试
  - `pdb`：Python 调试
  - `ros2 run rqt_console rqt_console`：日志查看
- **可视化工具**：
  - `ros2 run rqt_graph rqt_graph`：节点关系图
  - `rviz2`：3D 可视化

### 8.2 最佳实践

1. **节点设计**：
   - 每个节点专注于单一功能
   - 合理设置队列大小和 QoS 策略
   - 使用参数而非硬编码值

2. **通信设计**：
   - 合理选择通信方式（话题/服务/动作）
   - 定义清晰的消息结构
   - 考虑网络带宽和延迟

3. **错误处理**：
   - 使用 ROS2 的日志系统
   - 实现适当的错误恢复机制
   - 提供清晰的错误消息

4. **性能优化**：
   - 避免在回调中执行长时间操作
   - 使用定时器而非忙等待
   - 合理使用缓存

5. **代码组织**：
   - 遵循 ROS2 包结构规范
   - 模块化设计
   - 清晰的命名约定

## 9. 故障排除

### 9.1 常见问题

- **节点启动失败**：
  - 检查依赖是否安装
  - 检查环境变量是否正确设置
  - 查看日志输出

- **通信问题**：
  - 检查话题名称是否正确
  - 检查消息类型是否匹配
  - 使用 `ros2 topic list` 和 `ros2 node info` 诊断

- **构建失败**：
  - 检查 CMakeLists.txt 或 setup.py
  - 检查依赖项
  - 查看详细的构建日志

### 9.2 调试技巧

- **启用详细日志**：
  ```bash
  ros2 run my_package my_node --ros-args --log-level debug
  ```

- **使用 rqt_console**：
  ```bash
  ros2 run rqt_console rqt_console
  ```

- **使用 ros2 doctor**：
  ```bash
  ros2 doctor
  ```
