# SynapticSYS 用户指南

## 1. 简介

SynapticSYS 是一个基于 ROS2 的智能系统开发平台，提供了一套完整的工具链，用于构建、测试和部署基于 ROS2 的机器人和智能系统应用。本指南将帮助您快速上手使用 SynapticSYS 平台。

## 2. 安装

### 2.1 系统要求

- **ROS2**：Humble 或更高版本
- **Python**：3.8 或更高版本
- **支持的操作系统**：
  - Windows 10/11
  - macOS 10.15 或更高版本
  - Linux (Ubuntu 22.04+, Debian 11+, Fedora 36+)

### 2.2 安装步骤

1. **安装 ROS2**
   - 请参考 [ROS2 官方安装指南](https://docs.ros.org/en/humble/Installation.html) 安装 ROS2 Humble 或更高版本
   - 确保设置了正确的环境变量：
     ```bash
     source /opt/ros/humble/setup.bash
     ```

2. **安装依赖**
   ```bash
   # 安装 Colcon 构建工具
   sudo apt install python3-colcon-common-extensions python3-rosdep
   
   # 安装其他依赖
   pip install -r requirements.txt
   ```

3. **克隆仓库**
   ```bash
   git clone https://github.com/your-organization/SynapticSYS.git
   cd SynapticSYS
   ```

4. **安装 SynapticSYS**
   ```bash
   pip install -e .
   ```

5. **验证安装**
   ```bash
   synaptic --version
   ```

## 3. 基本概念

### 3.1 ROS2 工作区

SynapticSYS 使用标准的 ROS2 工作区结构：

- `src/`：存放 ROS2 包源代码
- `build/`：构建输出目录（自动生成）
- `install/`：安装目录（自动生成）
- `log/`：日志目录（自动生成）

### 3.2 ROS2 包

包是 ROS2 中的基本构建单元，每个包通常包含：

- **package.xml**：包的元数据和依赖声明
- **构建配置文件**：
  - C++ 包：`CMakeLists.txt`
  - Python 包：`setup.py` 或 `setup.cfg`
- **源文件**：
  - C++ 包：`src/` 和 `include/` 目录
  - Python 包：包同名目录

### 3.3 ROS2 依赖关系

ROS2 包的依赖关系在 `package.xml` 中声明：

- `depend`：运行时和构建时依赖
- `build_depend`：仅构建时依赖
- `build_export_depend`：构建导出依赖
- `exec_depend`：仅运行时依赖
- `test_depend`：测试依赖

## 5. 配置文件

### 5.1 ROS2 包配置

每个 ROS2 包必须包含 `package.xml` 文件：

```xml
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>my_package</name>
  <version>0.0.1</version>
  <description>My ROS2 package</description>
  <maintainer email="user@example.com">User</maintainer>
  <license>Apache License 2.0</license>
  
  <buildtool_depend>ament_cmake</buildtool_depend>
  
  <depend>rclcpp</depend>
  <depend>std_msgs</depend>
  
  <test_depend>ament_cmake_gtest</test_depend>
  <test_depend>rclcpp_test</test_depend>
  
  <export>
    <build_type>ament_cmake</build_type>
  </export>
</package>
```

### 5.2 CMake 配置

C++ 包的 `CMakeLists.txt` 示例：

```cmake
cmake_minimum_required(VERSION 3.8)
project(my_package)

if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  add_compile_options(-Wall -Wextra -Wpedantic)
endif()

find_package(ament_cmake REQUIRED)
find_package(rclcpp REQUIRED)
find_package(std_msgs REQUIRED)

add_executable(my_node src/my_node.cpp)
ament_target_dependencies(my_node rclcpp std_msgs)

install(TARGETS
  my_node
  DESTINATION lib/${PROJECT_NAME}
)

if(BUILD_TESTING)
  find_package(ament_cmake_gtest REQUIRED)
  add_gtest(test_my_node test/test_my_node.cpp)
  ament_target_dependencies(test_my_node rclcpp std_msgs)
endif()

ament_package()
```

### 5.3 Python 配置

Python 包的 `setup.py` 示例：

```python
from setuptools import setup

package_name = 'my_python_package'

setup(
    name=package_name,
    version='0.0.1',
    packages=[package_name],
    data_files=[
        ('share/ament_index/resource_index/packages',
            ['resource/' + package_name]),
        ('share/' + package_name, ['package.xml']),
    ],
    install_requires=['setuptools'],
    zip_safe=True,
    maintainer='User',
    maintainer_email='user@example.com',
    description='Python ROS2 package',
    license='Apache License 2.0',
    tests_require=['pytest'],
    entry_points={
        'console_scripts': [
            'my_python_node = my_python_package.my_node:main',
        ],
    },
)
```

## 6. 使用示例

### 6.1 基本 ROS2 开发流程

1. **创建工作区**
   ```bash
   mkdir -p ~/synapticsys_ws/src
   cd ~/synapticsys_ws
   ```

2. **创建 ROS2 包**
   ```bash
   # 创建 C++ 包
   cd src
   ros2 pkg create --build-type ament_cmake my_cpp_package
   
   # 或创建 Python 包
   ros2 pkg create --build-type ament_python my_python_package
   cd ..
   ```

3. **构建工作区**
   ```bash
   synaptic build
   ```

4. **设置环境**
   ```bash
   source install/setup.bash
   ```

5. **运行节点**
   ```bash
   ros2 run my_cpp_package my_node
   ```

6. **查看话题**
   ```bash
   ros2 topic list
   ```

7. **运行测试**
   ```bash
   synaptic test
   ```

### 6.2 高级使用场景

#### 6.2.1 使用 launch 文件

创建 `launch/my_launch.py` 文件：

```python
from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
    return LaunchDescription([
        Node(
            package='my_cpp_package',
            executable='my_node',
            name='my_node'
        ),
        Node(
            package='my_python_package',
            executable='my_python_node',
            name='my_python_node'
        ),
    ])
```

运行 launch 文件：

```bash
ros2 launch my_cpp_package my_launch.py
```

#### 6.2.2 使用参数

在 `config/my_config.yaml` 文件中定义参数：

```yaml
my_node:
  ros__parameters:
    param1: value1
    param2: 42
```

在 launch 文件中加载参数：

```python
from launch import LaunchDescription
from launch_ros.actions import Node
from launch_ros.substitutions import FindPackageShare
import os

def generate_launch_description():
    config = os.path.join(
        FindPackageShare('my_cpp_package').find('my_cpp_package'),
        'config',
        'my_config.yaml'
    )
    
    return LaunchDescription([
        Node(
            package='my_cpp_package',
            executable='my_node',
            name='my_node',
            parameters=[config]
        ),
    ])
```

#### 6.2.3 使用自定义消息

创建 `msg/MyMessage.msg` 文件：

```
int32 id
string name
float64 value
```

在 `CMakeLists.txt` 中添加消息生成：

```cmake
find_package(rosidl_default_generators REQUIRED)

rosidl_generate_interfaces(${PROJECT_NAME}
  "msg/MyMessage.msg"
  DEPENDENCIES std_msgs
)

ament_target_dependencies(my_node rosidl_default_runtime)
```

## 9. 常见问题

### 9.1 构建失败

如果构建失败，尝试以下解决方法：

1. 检查错误信息，通常会显示具体的失败原因
2. 清理构建目录：`synaptic clean`
3. 检查依赖关系是否正确声明
4. 尝试使用 `--verbose` 选项获取更详细的输出
5. 确保 ROS2 环境正确设置：`source /opt/ros/humble/setup.bash`

### 9.2 节点启动失败

如果节点启动失败，检查：

1. 环境变量是否正确设置：`source install/setup.bash`
2. 依赖包是否已安装
3. 节点名称是否冲突
4. 查看详细日志：`ros2 run rqt_console rqt_console`

### 9.3 通信问题

如果节点间通信有问题，检查：

1. 话题名称是否正确
2. 消息类型是否匹配
3. QoS 设置是否合适
4. 使用 `ros2 topic list` 和 `ros2 topic echo /my_topic` 检查话题

### 9.4 依赖解析问题

如果依赖解析失败：

1. 确保所有依赖包都已安装
2. 检查 `package.xml` 文件中的依赖声明是否正确
3. 运行 `rosdep install --from-paths src --ignore-src -r -y` 安装缺失的依赖
