## 📝 文档维护与贡献指南

### 维护规范
- **新增文档**：遵循 `NN-标题.md` 命名规范（NN 为两位数字序号）
- **更新时机**：重要决策后 24 小时内完成文档更新
- **版本记录**：在每个文档顶部记录更新日期和关键变更
- **交叉引用**：相关文档之间使用 Markdown 链接相互引用
- **语言一致性**：中英文版本保持同步更新
- **版本控制**：使用清晰、描述性的提交信息，遵循 Conventional Commits 规范

### 贡献流程
1. **Fork 并克隆**仓库
2. **创建分支**用于您的修改（例如：`doc-update/ros2-integration`）
3. **进行修改**：遵循现有样式和结构，确保ROS2相关内容的准确性
4. **审查您的工作**：确保内容准确、一致且格式良好
5. **提交拉取请求**：包含清晰的变更描述
6. **处理反馈**：及时回应审查意见
7. **合并**：获得批准后，您的变更将被合并到主分支

### 样式指南
- **语言**：使用清晰、简洁的中文；尽可能避免行话
- **结构**：遵循现有文档层次结构和编号系统
- **格式**：使用一致的 Markdown 格式（标题、列表、代码块）
- **链接**：内部文档使用相对链接；外部资源使用绝对链接
- **图片**：将图片放在 `images/` 子目录中；使用描述性文件名
- **表格**：使用一致的列对齐和格式

### 审查流程
- **自我审查**：提交前检查准确性、完整性和一致性
- **同行审查**：所有文档变更需要至少一次同行审查
- **技术审查**：架构相关变更需要架构团队的技术审查
- **批准**：变更由文档维护者批准
