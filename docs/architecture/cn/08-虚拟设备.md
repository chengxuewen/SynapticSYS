# 虚拟设备

虚拟设备（Virtual Device）是 SynapticSYS 中的硬件抽象层，基于ROS2开发，用于在开发与测试环境中模拟外设与设备行为，避免直接依赖真实硬件，提升开发效率与稳定性。

## 目标与作用

- 快速构建设备仿真环境，支持多种设备类型与协议。
- 提供统一的设备抽象与接口，使业务逻辑脱离具体硬件依赖。
- 支持消息回放与场景复现，提升测试覆盖与问题定位效率。

## 虚拟设备技术调研

### 行业概述

虚拟设备技术近年来发展迅速，主要驱动因素包括：

- **DevOps 实践**：需要一致的开发/测试环境
- **IoT 普及**：设备类型和协议数量不断增长
- **边缘计算**：边缘设备仿真需求
- **云集成**：连接物理世界与数字世界

#### 2. 虚拟化技术对比

| 技术类型 | 描述 | 使用场景 |
|----------|------|----------|
| **仿真** | 完整硬件模拟 | legacy 设备支持 |
| **虚拟化** | 硬件资源共享 | 资源优化 |
| **容器化** | 轻量级隔离 | 跨平台兼容性 |
| **数字孪生** | 实时设备镜像 | 预测性维护 |

#### 3. 行业解决方案

- **ROS2 Gazebo**：机器人仿真环境，与ROS2深度集成
- **ROS2 虚拟设备**：基于ROS2的设备仿真能力
- **QEMU**：硬件模拟器和虚拟机
- **Docker Devices**：容器设备虚拟化
- **Azure IoT Device Simulator**：基于云的设备仿真

### 技术挑战

1. **实时性能**：虚拟设备必须匹配真实设备的延迟
2. **协议兼容性**：支持多种通信协议（CAN、MODBUS 等）
3. **状态同步**：维护虚拟设备和真实设备之间的一致状态
4. **可扩展性**：支持成百上千的虚拟设备
5. **安全性**：保护设备通信和数据

## 本项目研发方向

### 核心目标

1. **统一抽象**：所有设备交互的单一 API
2. **无缝切换**：无需代码更改即可在虚拟设备和真实设备之间切换
3. **丰富仿真**：逼真的设备行为建模
4. **性能优化**：实时响应保证
5. **可扩展性**：轻松添加新设备类型和协议

## 技术路线图

### 短期（0-6 个月）

1. **核心 API 开发**
   - 定义统一设备接口
   - 实现基础设备管理器

2. **ROS2 集成**
   - 开发 ROS2 集成层
   - 实现 ROS2 节点和话题/服务

3. **基础设备支持**
   - USB 输入设备（键盘、鼠标、游戏手柄）
   - GPIO 数字 I/O
   - 简单虚拟传感器

4. **虚拟/真实切换**
   - 基于配置的设备选择
   - 热插拔检测和处理

### 中期（6-12 个月）

1. **高级协议支持**
   - CAN 总线仿真和支持
   - Modbus TCP/RTU 集成
   - 串口（RS-232/485）支持

2. **丰富仿真能力**
   - 基于物理的传感器模型
   - 执行器响应仿真
   - 数据录制和回放

3. **场景引擎**
   - 复杂交互场景定义
   - 测试用错误注入

### 长期（12+ 个月）

1. **AI 集成**
   - 基于机器学习的设备行为建模
   - 预测性仿真能力

2. **数字孪生支持**
   - 与物理设备实时同步
   - 基于云的设备镜像

3. **边缘计算**
   - 边缘设备仿真
   - 分布式设备管理

4. **生态系统扩展**
   - 第三方设备集成 SDK
   - 虚拟设备模型市场

## 使用场景

- 新功能验证：在没有真实设备时进行功能开发与集成测试。
- 异常注入：模拟网络抖动、设备故障、协议错误等场景。
- 性能评估：评估系统在高并发与高流量下的表现。

## 与生产环境的关系

- 统一接口与协议，保证仿真与生产一致性。
- 配置化切换，允许在仿真与真实设备之间快速切换。
- 可观测性与回放能力促进问题定位与复盘。
