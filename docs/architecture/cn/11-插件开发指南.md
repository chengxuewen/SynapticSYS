# 插件开发指南

SynapticSYS 基于 ROS2 的插件系统，允许开发者通过编写自定义插件来扩展其功能。本指南将引导您完成创建和注册自己的 ROS2 插件的过程。

## 插件系统架构

### 核心概念

- **插件接口（Plugin Interface）**：定义插件必须实现的接口
- **插件实现（Plugin Implementation）**：接口的具体实现
- **插件加载器（Plugin Loader）**：负责发现、加载和实例化插件
- **插件管理器（Plugin Manager）**：管理插件的生命周期

### ROS2 插件系统

SynapticSYS 使用 ROS2 的 `pluginlib` 库作为插件系统的基础：

1. **基于 C++ 的插件系统**：适用于性能关键型组件
2. **基于 Python 的插件系统**：适用于快速开发和原型设计
3. **基于 JavaScript 的插件系统**：适用于 Web 前端和脚本开发
4. **统一的插件发现机制**：通过 `package.xml` 声明插件

## 入门指南

### C++ 插件开发

#### 插件结构

C++ 插件通常按以下方式组织：

```
my_plugin_package/
├── package.xml
├── CMakeLists.txt
├── include/my_plugin_package/
│   └── my_plugin_interface.hpp
└── src/
    ├── my_plugin_interface.cpp
    └── my_plugin_implementation.cpp
```

#### 定义插件接口

```cpp
// include/my_plugin_package/my_plugin_interface.hpp
#ifndef MY_PLUGIN_INTERFACE_HPP
#define MY_PLUGIN_INTERFACE_HPP

#include <pluginlib/class_list_macros.hpp>

namespace my_plugin_package
{
  class MyPluginInterface
  {
  public:
    virtual ~MyPluginInterface() = default;
    virtual void initialize() = 0;
    virtual void execute() = 0;
    virtual std::string get_name() const = 0;
  };

}  // namespace my_plugin_package

// 声明插件接口
PLUGINLIB_DECLARE_CLASS(
  my_plugin_package,
  MyPluginInterface,
  my_plugin_package::MyPluginInterface,
  my_plugin_package::MyPluginInterface
)

#endif  // MY_PLUGIN_INTERFACE_HPP
```

#### 实现插件

```cpp
// src/my_plugin_implementation.cpp
#include "my_plugin_package/my_plugin_interface.hpp"

namespace my_plugin_package
{
  class MyPluginImplementation : public MyPluginInterface
  {
  public:
    MyPluginImplementation() {}
    
    void initialize() override
    {
      // 初始化插件
    }
    
    void execute() override
    {
      // 执行插件逻辑
    }
    
    std::string get_name() const override
    {
      return "my_plugin_implementation";
    }
  };

}  // namespace my_plugin_package

// 注册插件
PLUGINLIB_EXPORT_CLASS(
  my_plugin_package::MyPluginImplementation,
  my_plugin_package::MyPluginInterface
)
```

#### 配置 CMakeLists.txt

```cmake
cmake_minimum_required(VERSION 3.8)
project(my_plugin_package)

if(CMAKE_COMPILER_IS_GNUCXX OR CMAKE_CXX_COMPILER_ID MATCHES "Clang")
  add_compile_options(-Wall -Wextra -Wpedantic)
endif()

find_package(ament_cmake REQUIRED)
find_package(pluginlib REQUIRED)
find_package(rclcpp REQUIRED)

add_library(
  ${PROJECT_NAME}
  SHARED
  src/my_plugin_interface.cpp
  src/my_plugin_implementation.cpp
)

ament_target_dependencies(
  ${PROJECT_NAME}
  pluginlib
  rclcpp
)

target_include_directories(
  ${PROJECT_NAME}
  PUBLIC
  $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}/include>
  $<INSTALL_INTERFACE:include>
)

install(
  TARGETS ${PROJECT_NAME}
  DESTINATION lib
)

install(
  DIRECTORY include/
  DESTINATION include
)

pluginlib_export_plugin_description_file(my_plugin_package plugins.xml)

ament_package()
```

#### 创建插件描述文件

创建 `plugins.xml` 文件：

```xml
<!-- plugins.xml -->
<library path="libmy_plugin_package">
  <class type="my_plugin_package::MyPluginImplementation" base_class_type="my_plugin_package::MyPluginInterface">
    <description>我的插件实现</description>
  </class>
</library>
```

#### 更新 package.xml

```xml
<!-- package.xml -->
<?xml version="1.0"?>
<?xml-model href="http://download.ros.org/schema/package_format3.xsd" schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
  <name>my_plugin_package</name>
  <version>0.0.1</version>
  <description>插件包示例</description>
  <maintainer email="user@example.com">User</maintainer>
  <license>Apache License 2.0</license>
  
  <buildtool_depend>ament_cmake</buildtool_depend>
  
  <depend>pluginlib</depend>
  <depend>rclcpp</depend>
  
  <export>
    <build_type>ament_cmake</build_type>
    <my_plugin_package plugin="${prefix}/plugins.xml"/>
  </export>
</package>
```

## 加载和使用插件

### C++ 插件加载

```cpp
#include "my_plugin_package/my_plugin_interface.hpp"
#include <pluginlib/class_loader.hpp>

int main(int argc, char **argv)
{
  // 初始化 ROS2
  rclcpp::init(argc, argv);
  
  try {
    // 创建插件加载器
    pluginlib::ClassLoader<my_plugin_package::MyPluginInterface> plugin_loader(
      "my_plugin_package",
      "my_plugin_package::MyPluginInterface"
    );
    
    // 加载插件
    std::shared_ptr<my_plugin_package::MyPluginInterface> plugin = 
      plugin_loader.createInstance("my_plugin_package::MyPluginImplementation");
    
    // 使用插件
    plugin->initialize();
    plugin->execute();
    std::cout << "Plugin name: " << plugin->get_name() << std::endl;
    
  } catch (const pluginlib::PluginlibException &ex) {
    std::cerr << "Failed to load plugin: " << ex.what() << std::endl;
  }
  
  // 关闭 ROS2
  rclcpp::shutdown();
  return 0;
}
```

### Python 插件加载

```python
from pluginlib import PluginLoader

# 创建插件加载器
loader = PluginLoader(
    lib_name='my_python_plugin',
    base_class_type='my_python_plugin.my_plugin_interface.MyPluginInterface',
    plugin_xml_path='my_python_plugin/plugins.xml'
)

# 获取可用插件
available_plugins = loader.get_declared_classes()
print(f"Available plugins: {available_plugins}")

# 加载插件
plugin = loader.create_instance('my_python_plugin.my_plugin_implementation.MyPluginImplementation')

# 使用插件
plugin.initialize()
plugin.execute()
print(f"Plugin name: {plugin.get_name()}")
```

## 插件发现机制

### ROS2 插件发现

ROS2 的插件发现基于以下机制：

1. **package.xml 声明**：在 `<export>` 标签中声明插件
2. **plugins.xml 描述**：详细描述插件的实现
3. **ament 索引**：构建时生成的插件索引

### 自定义插件发现

除了 ROS2 标准插件系统外，SynapticSYS 还提供了自定义插件发现机制：

- **基于配置文件的发现**：通过 YAML/JSON 配置文件声明插件
- **基于目录的发现**：自动扫描指定目录中的插件
- **动态插件加载**：运行时加载插件

## 插件使用示例

### 在 SynapticSYS 中使用插件

#### 配置插件

在 `package.xml` 中声明插件依赖：

```xml
<depend>my_plugin_package</depend>
```

#### 加载插件

```cpp
// 在节点中加载插件
#include "my_plugin_package/my_plugin_interface.hpp"
#include <pluginlib/class_loader.hpp>

class MyNode : public rclcpp::Node
{
private:
  std::shared_ptr<pluginlib::ClassLoader<my_plugin_package::MyPluginInterface>> plugin_loader_;
  std::shared_ptr<my_plugin_package::MyPluginInterface> plugin_;
  
public:
  MyNode() : Node("my_node")
  {
    // 创建插件加载器
    plugin_loader_ = std::make_shared<pluginlib::ClassLoader<my_plugin_package::MyPluginInterface>>(
      "my_plugin_package",
      "my_plugin_package::MyPluginInterface"
    );
    
    // 加载配置的插件
    std::string plugin_name = this->declare_parameter("plugin_name", "my_plugin_package::MyPluginImplementation");
    
    try {
      plugin_ = plugin_loader_->createInstance(plugin_name);
      plugin_->initialize();
      RCLCPP_INFO(this->get_logger(), "Loaded plugin: %s", plugin_->get_name().c_str());
    } catch (const pluginlib::PluginlibException &ex) {
      RCLCPP_ERROR(this->get_logger(), "Failed to load plugin: %s", ex.what());
    }
  }
  
  void use_plugin() {
    if (plugin_) {
      plugin_->execute();
    }
  }
};
```

## 调试插件

### 常见问题

1. **插件未找到**：
   - 检查 `package.xml` 中的插件声明
   - 验证 `plugins.xml` 文件路径正确
   - 确保插件库已正确构建和安装

2. **插件加载失败**：
   - 检查插件依赖是否满足
   - 验证插件实现是否正确
   - 查看详细的错误消息

3. **插件初始化失败**：
   - 检查插件的 `initialize()` 方法
   - 验证插件的依赖项是否可用

### 调试工具

1. **ROS2 工具**：
   - `ros2 pkg list`：检查包是否安装
   - `ros2 node info`：查看节点信息
   - `ros2 topic list`：查看话题列表

2. **插件库工具**：
   - `pluginlib-list`：列出可用的插件
   - `ldd`（Linux）：检查库依赖
   - `otool -L`（macOS）：检查库依赖
